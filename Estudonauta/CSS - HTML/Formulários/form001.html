Meu primeiro formulário

  • Todo formulário deve estar dentro da tag <form>
  • Para criar um formulário de texto use a tag <input type="text" name="" id="">
    • "name" e "id" são obrigatórios
      • "Name" - Mais útil para HTML e PHP
      • "Id" - Mais útil para JavaScript
  • Para criar um "botão" de enviar use a tag <input type="submit" value="">
    • O parâmetro "value" será a o "nome" do seu botão de enviar
    • Exemplos:
  • Para desligar a sugestão dentro das caixas de texto, basta usar o parâmetro autocomplete="off" ao lado da tag <form>
    • Importante ressaltar que desligar a sugestão não é obrigatório, desligue caso esteja incomodando ou se não for a proposta do seu formulário
  • Para mandar os dados do cadastro ao clicar no botão de enviar (input:submit) temos que adicionar o parâmetro action="nome do código" ao lado da tag <form>
    • Esse "lugar" que vamos enviar os dados colocados pelo usuário deve ser para outro código e esse código deverá ser feito em php, java servidor ou outra linguagem de programação da sua escolha, neste caso vamos usar php
  • Para fazer a ligação entre o "Nome" e sua respectiva caixa de texto devemos usar as labels(etiquetas)
    • Envelopar o nome da caixa de texto na tag <label for="">, dentro de "for" colocar o id da caixa de texto
  • Method (dentro de <form>):
    • GET - Configuração padrão que deixa os dados enviados pelo usuário na URL.
      • Usar quando os dados não forem sensíveis, por exemplo, ao pedir nome, peso, coisas "fúteis" e quando quiser que a URL seja compartilhável
      • NÃO usar para formulários com envio de fotos ou qualquer outro tipo de arquivo
      • Ao enviar dados por GET você tem um limite de 3000 bytes
    • POST - Esconde os dados enviados pelo usuário na URL.
      • Para dados sensíveis, como senhas, dados de cartão, endereço, CEP, entre outros, para dados com mais de 3000 bytes, para fazer wenvio de arquivos
      • Os dados podem estar escondidos da URL, porem ainda conseguimos vê-los através das configurações do DevTools, ao entrar em Network e depois em Headers. Então os dados não estão 100% seguros e muito menos criptografados.
